1. Overview

ESP32JTAG is a compact Wi-Fi-enabled debug instrument built around the ESP32-S3. It combines several tools that embedded engineers typically need as separate pieces of hardware into one device accessible entirely through a web browser:

  • MCU JTAG / SWD debugger — based on Black Magic Probe (BMP), exposes a GDB server on TCP port 4242
  • 16-channel logic analyzer — up to 264 MHz sample rate, browser-based waveform viewer
  • Signal generator (Port D) — drives known signals onto the target circuit for stimulus and loopback testing: FPGA free-running counter at 132 MHz (bits [3:0] or [7:4]) and software square waves at 125 / 250 / 500 / 1000 Hz; applied instantly from the web UI without rebooting
  • FPGA programmer — JTAG and SPI configuration, Xilinx Virtual Cable (XVC) server on TCP port 2542
  • UART web terminal — browser serial console with configurable baud/parity

All functions share the same four physical connector ports (A–D). Each port is independently configured for the function you need.

2. Quick Start

  1. Power on. Connect ESP32JTAG to a 5 V USB-C source. The small LCD turns on and shows the current mode and IP address.
  2. First boot — connect to the device AP. On first boot the device broadcasts a Wi-Fi access point:
    • SSID: esp32jtag_XXYY  (XXYY = last 2 bytes of MAC, shown on LCD)
    • Password: esp32jtag
  3. Open the web interface. Navigate to https://192.168.4.1 in your browser. Accept the self-signed TLS certificate warning. Log in with admin / admin.
  4. Configure Wi-Fi (optional). On the System Configurations tab, set Wi-Fi Mode to ST (Station Mode), enter your network SSID and password, then click Save settings & Restart. The LCD will show the new IP address after reconnection.
  5. Configure the ports. Set Port A–D to the functions you need and click Save settings & Restart.
  6. Use the tools. Access the Logic Analyzer from the top tab. Connect GDB or openFPGALoader to the device IP on the respective TCP port.
Tip: The LCD always shows the current IP address, Wi-Fi mode, and active port functions. Use it as a quick status check without opening a browser.

3. System Configuration

The System Configurations tab is the main settings page. Changes take effect after clicking Save settings & Restart, which saves all settings to flash and reboots the device.

Target IO Voltage

Sets the voltage output on the Vtarget pin (Port B, when configured as Vtarget + UART + SReset). Match this to your target MCU's IO voltage.

OptionVoltageTypical use
3.3V (default)3.30 VMost 3.3V MCUs, FPGAs
2.5V2.50 VSome FPGAs (e.g. older Xilinx devices)
1.8V1.80 VLow-voltage MCUs, modern FPGAs

1.2V1.20 VUltra-low-voltage targets
Warning: Always set the correct Target IO Voltage before connecting to your target. Applying a higher voltage than the target supports can damage it. Default after factory reset is 3.3 V.

SRESET Polarity & Pulse Width

These two controls configure the reset pulse that is sent when you click Reset Target (or call POST /api/reset_target). They take effect immediately when you click Apply SRESET Settings, Port D Settings and Target IO Voltage — no reboot is required.

ControlDefaultDescription
SRESET Polarity Negative (active LOW) Negative (active LOW) — idle HIGH, pulse LOW. This is the most common convention for microcontroller RESET# pins.
Positive (active HIGH) — idle LOW, pulse HIGH. Use for targets that require an active-high reset signal.
Pulse Width 100 ms Duration of the reset pulse in milliseconds (1–5000). Most targets only need a few milliseconds; 100 ms provides reliable startup margin.
Note: SRESET settings are applied in RAM and are not saved to flash. They revert to the defaults (active LOW, 100 ms) after a reboot.

Apply SRESET Settings, Port D Settings and Target IO Voltage

This button applies three groups of settings to the hardware immediately — no reboot required:

  1. SRESET Polarity & Pulse Width — sent to /api/sreset_config; takes effect on the next Reset Target action.
  2. Port D output mode — switches Port D between Counter Lo/Hi or GPIO Direct frequency modes instantly.
  3. Target IO Voltage — note: voltage changes still require Save settings & Restart to take effect; clicking this button alone is not sufficient.
Note: Port D output modes (Counter Lo/Hi, GPIO Direct) are transient — not written to flash. They revert to Logic Analyzer after a reboot. Only Logic Analyzer and FPGA XVC are persistent Port D modes.

Reset Target

The Reset Target button sends a configurable reset pulse on Port B pin 3 (SReset line), immediately resetting the connected target MCU. The pulse polarity and width are controlled by the SRESET Polarity and Pulse Width settings above (default: active LOW, 100 ms).

Requires: Port B must be configured as Vtarget + UART + SReset for this button to work.

Set to Default / Factory Reset

The Set to default button erases all saved settings from flash and reboots. After reset:

  • Wi-Fi returns to AP Mode with the default SSID and password
  • All port configurations return to Logic Analyzer
  • Target IO Voltage returns to 3.3 V
  • Web credentials return to admin / admin
  • UART settings return to 115200 / 8N1

Port D — Signal Generation, FPGA, or Logic Analyzer

Port D doubles as a signal generator for the target system. When set to a signal generation mode, all four Port D pins drive a known signal that can be used as clock injection, functional excitation, or loopback stimulus for the target circuit — without any external signal generator. Apply instantly via Apply SRESET Settings, Port D Settings and Target IO Voltage; no reboot needed.

Note: When a port is assigned to a specific function (SWD, UART, FPGA XVC, etc.), its four pins are no longer available as logic analyzer channels. Only ports still set to Logic Analyzer contribute channels to the waveform viewer.

5. MCU Debugging (Black Magic Probe / GDB)

ESP32JTAG embeds the Black Magic Probe (BMP) firmware. It exposes a GDB server over TCP so any GDB client (arm-none-eabi-gdb, VSCode Cortex-Debug, etc.) can connect without a USB cable.

Setup

  1. Set Port A to SWD or JTAG (or Port C to SWD/JTAG) in System Configurations.
  2. Click Save settings & Restart.
  3. Wire the SWD/JTAG signals from Port A (or C) to your target MCU.
  4. Connect your GDB client to <device-ip>:4242.

GDB Connection Example

arm-none-eabi-gdb your_firmware.elf (gdb) target extended-remote 192.168.1.x:4242 (gdb) monitor swdp_scan # scan for SWD targets (gdb) attach 1 (gdb) load # flash the binary (gdb) continue

VSCode / Cortex-Debug (launch.json)

{ "type": "cortex-debug", "request": "attach", "servertype": "external", "gdbTarget": "192.168.1.x:4242", "executable": "${workspaceFolder}/build/your_firmware.elf" }
Tip: The LCD displays GDB:4242 on the port status line as a reminder of the GDB server port.

Supported Targets

BMP supports a wide range of ARM Cortex-M and Cortex-A targets including STM32, nRF52, RP2040, LPC, SAM, and many others. Run monitor jtag_scan or monitor swdp_scan in GDB to detect connected targets automatically.

Trigger

Each channel can be independently configured with one of the following trigger modes:

ModeDescription
DisabledChannel does not participate in triggering
RisingTrigger on low→high transition
FallingTrigger on high→low transition
CrossingTrigger on any edge
HighTrigger when signal is high
LowTrigger when signal is low

Trigger mode (AND / OR) controls how multiple channel triggers combine:

  • OR: capture starts when any of the configured channels fires.
  • AND: capture starts only when all configured channels fire simultaneously.

Trigger Position sets how much pre-trigger data is retained (0–100%). At 50% the trigger event falls in the middle of the capture window.

Controls

  • Start Capture — arms the trigger and begins streaming; waveform updates live.
  • Instant Capture — captures immediately without waiting for a trigger.
  • Test Signal: ON/OFF — enables an internal test signal on one channel for verifying connectivity without an external source.
  • Measure: ON/OFF — enables cursor-based time measurement between two points.
  • Save / Load Capture — export or import waveform data as a JSON file.

Settings (gear icon)

Open the Settings popup to adjust sample rate, trigger position, trigger enable/disable, trigger mode (OR/AND), capture internal test signal, and per-channel trigger types. Settings are applied immediately on Save.

7. FPGA Programming

ESP32JTAG supports FPGA configuration via JTAG and SPI, and exposes a Xilinx Virtual Cable (XVC) server for use with standard tools.

XVC (Xilinx Virtual Cable)

  1. Set Port D to FPGA XVC in System Configurations and restart.
  2. Connect the JTAG pins of Port D to your FPGA's JTAG header.
  3. Use any XVC-compatible tool and point it to <device-ip>:2542.

Example with openFPGALoader:

openFPGALoader --cable xvc-client --xvc-url tcp:192.168.1.x:2542 \ --fpga-part xc7a35t bitstream.bit
Tip: The LCD shows XVC:2542 when Port D is in FPGA XVC mode.

8. UART Web Terminal

The UART Web Terminal tab provides a browser-based serial console bridged to Port B's UART pins.

Hardware Setup

  1. Set Port B to Vtarget + UART + SReset in System Configurations.
  2. Click Save settings & Restart.
  3. Connect Port B TX → Target RX, Port B RX → Target TX.
  4. Vtarget will supply the configured Target IO Voltage to the target.

UART Settings

Configure baud rate, data bits, stop bits, and parity to match your target device, then click Save UART Settings.

ParameterOptionsDefault
Baud Rate9600 / 19200 / 38400 / 57600 / 74880 / 115200 / 230400 / 460800 / 921600115200
Data Bits5 / 6 / 7 / 88
Stop Bits1 / 1.5 / 21
ParityNone / Even / OddNone

Connecting

  • Connect UART — opens a WebSocket to Port B UART; shows received data and allows sending.
  • Connect ESP32 Log Monitor — connects to the internal ESP32JTAG debug log stream (useful for firmware diagnostics).
  • Disconnect — closes the active connection.
Note: UART settings are saved to flash separately from the main System Configuration. Use the Save UART Settings button on the UART tab to persist baud/parity changes.

9. Firmware Update

Two release binaries are produced by every build. The filename encodes version, UTC build timestamp, and git commit so you can always identify which is newer:

File suffixUse forExample
_ota.bin Wireless OTA update (primary method — no USB cable needed) esp32jtag_v0.1.0_20260325_085159_c14c3fe_ota.bin
_full.bin Full flash via USB (factory / first-time programming) esp32jtag_v0.1.0_20260325_085159_c14c3fe_full.bin

OTA Update (wireless)

  1. Download the _ota.bin file for the target release.
  2. Open https://<device-ip>/ota_upload in your browser.
  3. Drag and drop the _ota.bin file onto the upload area, or click to browse.
  4. Click Upload & Update. A progress bar shows upload status.
  5. The device reboots automatically into the new firmware when the upload completes.
Warning: Do not power off the device during an OTA update. If the update fails, the device retains the previous firmware and you can retry.

USB Flash (factory / first-time programming)

Use the _full.bin file with esptool. This writes the complete flash image (bootloader + partition table + firmware) from offset 0x0:

pip install esptool   # one-time install

esptool.py --chip esp32s3 -p /dev/ttyUSB0 -b 460800 \
  --before default_reset --after hard_reset \
  write_flash 0x0 esp32jtag_v<VER>_<DATE>_<GIT>_full.bin

On Windows replace /dev/ttyUSB0 with the appropriate COM port (e.g. COM3).

Recovery Mode — firmware crash / cannot flash normally

If the firmware has crashed or is stuck in a boot loop and normal flashing fails, force the ESP32-S3 into ROM download mode using the BOOT0 button:

  1. Power off the device (disconnect USB).
  2. Hold the BOOT0 button (SW1, GPIO0) and keep it held.
  3. Power on (reconnect USB) while still holding BOOT0.
  4. Release BOOT0 after about one second.

The device is now in ROM download mode. The LCD shows no output — this is normal. Flash the _full.bin file using the esptool command above. The device will boot normally once flashing completes.

Note: Recovery mode requires the _full.bin (merged) file — not the _ota.bin file. OTA upload is not available in recovery mode because the web interface does not start.

11. Troubleshooting

Cannot connect to the web interface

  • Check the IP address shown on the LCD.
  • In Station Mode, ensure your computer is on the same Wi-Fi network.
  • In AP Mode, connect your computer to the esp32jtag_XXYY SSID.
  • The web server uses HTTPS — your browser will warn about a self-signed certificate; accept and continue.
  • Try a hard refresh (Ctrl+Shift+R / Cmd+Shift+R) or an incognito window.

Forgot Wi-Fi password / device unreachable

If you cannot reach the web interface at all, perform a factory reset:

  • Hold the device's BOOT button while pressing and releasing RESET, then release BOOT.
  • Or, if the web interface is accessible, use Set to default on the System Configurations page.

After reset the device returns to AP Mode with default credentials.

GDB cannot connect to port 4242

  • Confirm Port A (or C) is set to SWD or JTAG (not Logic Analyzer) and the device has restarted.
  • Verify the device IP using the LCD or ping <ip>.
  • Check your firewall is not blocking TCP port 4242.
  • Run monitor swdp_scan or monitor jtag_scan after attaching to detect the target.

Logic analyzer shows no data / all zeros

  • Ensure at least one port is set to Logic Analyzer.
  • Enable the internal Test Signal and run Instant Capture to verify the analyzer path works.
  • Check the trigger settings — if a trigger is armed but never fires, the capture will wait indefinitely. Use Instant Capture to bypass triggering.
  • Confirm the probe ground is connected to the target ground.

UART terminal shows no output

  • Port B must be set to Vtarget + UART + SReset.
  • Verify TX/RX wiring is not swapped (Port B TX → Target RX; Port B RX → Target TX).
  • Confirm baud rate matches the target device.
  • Check that Target IO Voltage is set correctly — a mismatch can prevent signal detection.

FPGA not detected via XVC

  • Port D must be set to FPGA XVC and the device restarted.
  • Verify JTAG wiring: TCK, TMS, TDI, TDO from Port D to the FPGA JTAG header.
  • Confirm the tool is connecting to <device-ip>:2542.

OTA update fails

  • Ensure the .bin file is a valid ESP32-S3 firmware image.
  • Keep the device and your computer on the same network with a stable connection.
  • If the upload stalls, refresh the page — the previous firmware is still intact.
